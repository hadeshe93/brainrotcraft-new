# 游戏聚合站技术规划草稿

我要开发并上线一个 H5 游戏聚合站，域名为 `gamesramp.com`，以下是我目前的规划。

## 页面及模块分析

面向用户页面按职能划分：

- 首页
- 聚合列表页
  - Categories
  - Tags
- 游戏列表页
  - All Games
  - 特色列表：Hot、Popular、Best
  - 常规分类：Action、Puzzle、...
- 游戏详情页：具体游戏

面向站点管理员页面按职能划分：

- 游戏管理列表页
- 游戏分类管理列表页
- 游戏评论管理列表页

## 首页

参考：https://www.gameflare.com/

### 聚合列表页

梳理：

- Categoris、Tags
  - 用途：展示所有的分类或者是标签，从 SEO 角度来看是将各个分类和标签入口给到爬虫
  - 组成：标题、描述、全列表展示（不需要分页组件）

### 游戏列表页

梳理：

- All Games
  - 用途：展示所有游戏给用户选择，从 SEO 角度来看增加站点停留时长，增加互动
  - 组成：标题、描述、游戏列表、分页组件
- 特色列表：Hot、Popular、Best
  - 用途：从不同角度展示给用户目前可能关心的趋势
  - 组成：标题、描述、游戏列表、分页组件
- 常规分类：Action、Puzzle、...
  - 用途：按游戏分类，给用户展示特定分类下的游戏列表
  - 组成：标题、描述、游戏列表、分页组件

### 游戏详情页

- 用途：承载具体游戏，最终用户玩游戏的页面
- 组成：
  - 游戏信息：名称、缩略图、iframe 内嵌游戏资源
  - 社交：Upvote、Downvote、收藏、分享、评分
  - 详情介绍：玩法内容、分类（可能有多个）
  - 用户评论列表
  - 热点游戏模块：有限数的列表（不用分页）、「更多」链接
  - 可能喜欢模块：有限数的列表（不用分页）、「更多」链接

### 游戏管理列表页

- 用途：管理所有游戏，包括增删改查、上架下架状态管理
- 组成：
  - 顶部工具栏：新增、批量上下架、批量删除
  - 中间表格方式呈现和管理，最右边一列是操作列
  - 底部分页组件

### 分类管理列表页

- 用途：管理所有分类，包括增删改查
- 组成：
  - 顶部工具栏：新增、批量删除
  - 中间表格方式呈现和管理，最右边一列是操作列
  - 底部分页组件

### 评论管理列表页

- 用途：管理所有用户评论，包括增删改查
- 组成：
  - 顶部工具栏：新增、批量删除
  - 中间表格方式呈现和管理，最右边一列是操作列
  - 底部分页组件

## 数据表设计

注意：

1. 本项目借用之前其他项目的沉淀，有部分数据表继承下来了，没有用到的也可以不用的，详见 @src/db/schema.ts
2. 本项目使用的数据库是 Cloudflare D1，驱动是 `drizzle-orm`，详见 @package.json

为了方便规划，我用了 TypeScript 的方式来表示数据表结构了：

```typescript
enum ERowStatus {
    Active,
    Inactive,
    Deleted, // 软删除
}
interface RowBase {
    id: number; // 数据表自增 ID
    uuid: string; // 唯一 UUID
    row_status: ERowStatus; // 数据状态
    created_at: number; // 创建时间戳，秒级
    updated_at: number; // 更新时间戳，秒级
    deleted_at:
}
interface SeoBase {
    metadata_title: string; // 元标题
    metadata_description: string; // 元描述
}
// 游戏详情
interface Detail extends RowBase {
    name: string; // 游戏名称
    thumbnail: string // 缩略图 url
    source: string // 资源地址 url
    interact: number; // 交互次数
    score: number; // 评分
    upvote: number; // 赞成数
    downvote: number; // 反对数
    save: number; // 收藏数
    share: number; // 分享数
    categories: string[]; // 所属的分类 UUID 列表
    tags: string[]; // 所属的标签 UUID 列表
    created_at: number; // 创建时间戳
    updated_at: number; // 更新时间戳
}
// 游戏评论
interface Comment extends RowBase {
    content: string; // 评论内容
    user_uuid: string; // 用户 UUID
}
// 符合 SEO 实践的游戏介绍
interface Introduction extends RowBase, SeoBase {
    content: string; // Markdwon 格式的游戏详情和玩法介绍，长文本
}
// 符合 SEO 实践的分类
interface Category extends RowBase, SeoBase {
    name: string; // 分类名称
    content: string; // Markdown 格式的内容，长文本
}
// 符合 SEO 实践的标签
interface Tag extends RowBase, SeoBase {
    name: string; // 标签名称
    content: string; // Markdown 格式的内容，长文本
}
// 符合 SEO 实践的特性
interface Featured extends RowBase, SeoBase {
    name: string;// 特性名称，例如：Hot、New 等等
    content: string; // Markdown 格式的内容，长文本
    detail_uuid: string[]; // 反向关联到具体 detail 下的 uuid，会按一定时间周期来滚动更新，比如 Hot 是每周根据上报数据来进行排序，选中前 N 名，然后保存其 uuid；而 New 则是每天更新一次；
}
```
